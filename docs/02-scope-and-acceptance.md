# Escopo e critérios de aceite

## Must have

1. Usuário consegue conectar uma wallet devnet.
2. Backend consegue ativar o acesso TxLINE com a wallet correta.
3. App lista uma partida e seu estado.
4. Usuário abre um único mercado de finalizações.
5. Backend rejeita dado stale, rede incompatível e request duplicado.
6. Posição fica registrada em devnet/paper mode.
7. Resultado oficial TxLINE fecha o mercado.
8. App mostra transaction signature, settlement ID e proof envelope.

## Não avançar antes de validar

- autenticação real;
- leitura real de evento de score/stat;
- mercado aberto/fechado;
- settlement feliz;
- settlement rejeitado;
- prova visível no demo.
